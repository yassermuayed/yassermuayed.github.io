# Scalable Vector Graphics

Scalable Vector Graphics (SVG) is an XML-based markup language for describing two-dimensional based vector graphics. SVG is defined by the W3C web standers [SVG 1.1 Second Edition Recondition](https://www.w3.org/TR/SVG11/).

Creating SVG files is very easy since it is only a text file that can be edited in any text editor and viewed in any web browser. There is a [small boilerplate snippet](./svg_file_boilerplate.md) to use in all your files.

SVG is not to meant to be written directly instead there are plenty of tools that are used to do so like Inkscape. However authoring and generating SVGs with coding is a different story. Alongside Knowledge of the language you will need to use libraries and tools. Here I keep a [list of tools and libraries](./svg_tools_and_libraries.md) that I use.

To be able to work with SVGs we will take a quick look at all the available elements for us to use. In [SVG Elements Catagories](./svg_elements_catagories.md) I listed all the elements I cloud find around the web and grouped them in a logical groups based on their use case.
